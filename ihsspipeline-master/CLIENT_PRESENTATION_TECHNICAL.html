BatchJobScheduler.java - Job Processing Loop
@Scheduled(fixedDelayString = "${batch.scheduler.interval-ms:5000}") public void pollAndDispatchJobs() { if (!properties.isEnabled()) return; // Get up to 3 queued jobs List<ReportJobEntity> queuedJobs = jobRepository.findTopQueuedJobs(3); for (ReportJobEntity job : queuedJobs) { // Atomically claim job using optimistic locking jobQueueService.markJobAsProcessing(job.getJobId()) .ifPresent(claimedJob -> batchJobExecutor.execute(() -> backgroundProcessingService.processJob(claimedJob.getJobId()) ) ); } }

üîÑ Optimistic Locking

Jobs use a version field to prevent race conditions. When multiple scheduler instances try to claim the same job, only one succeeds - others receive an empty Optional and skip to the next job.

Section 05

Security Model

JWT authentication, RBAC, and county-level data isolation

üîë

JWT Authentication

Keycloak issues JWT tokens containing user identity, roles, and custom claims (countyId). Tokens validated on every request.

üë•

Role-Based Access

6+ distinct roles with different permissions. Each role sees only authorized data with appropriate field masking applied.

üèõÔ∏è

County Isolation

Users restricted to their assigned county. County ID extracted from JWT - no fallbacks, no cross-county access.

üìã

Audit Logging

All data access operations logged via EventService. Complete audit trail for compliance and security monitoring.

üë§ User Roles & Access Levels

Role Scope Data Access Report Access
CENTRAL_WORKER State-wide All counties, all records All report types, minimal masking
DISTRICT_WORKER District Assigned district only District reports, some masking
COUNTY_WORKER County Assigned county only County reports, moderate masking
SUPERVISOR County Assigned county, team data Team reports, some masking
CASE_WORKER County Assigned cases only Case reports, standard masking
PROVIDER / RECIPIENT Self Own records only Personal reports, heavy masking
JWT Token Claims Structure
{ "sub": "user-uuid-12345", "preferred_username": "john.smith", "realm_access": { "roles": ["COUNTY_WORKER", "SUPERVISOR"] }, "countyId": "Los Angeles", // Custom claim for county isolation "districtId": "DISTRICT_SOUTH", // Optional district assignment "exp": 1702089600, // Token expiration "iss": "http://keycloak:8080/realms/cmips" }

üö´ No Fallback Policy

If countyId is missing from JWT token, the request FAILS. The system does not attempt to guess or default the county - this ensures data isolation is never compromised.
