How It Works

  • BatchJobScheduler polls for queued jobs every 5 seconds
  • Jobs are prioritized by priority level, then by creation time (oldest first)
  • BackgroundProcessingService processes each job asynchronously
  • Progress updates are stored in PostgreSQL and available via API
  • Failed jobs include detailed error messages for troubleshooting
  • Job dependencies are managed via JobDependencyService
Poll Interval: 5s Max Jobs/Poll: Configurable Thread Pool: Async Executor
