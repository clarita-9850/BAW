üîÑ Data Flow Workflow

The system processes timesheet data through a sophisticated 5-stage pipeline with role-based access control and advanced field masking.

5-Stage Data Processing Pipeline

1. Role Validation
2. Rules Engine Trigger
3. Query Building
4. Data Fetching
5. Field Masking Application

Pipeline Components

üîß Core Services:
  • RoleValidationService: Validates user role and permissions
  • RulesEngineService: Determines data access patterns and filtering criteria
  • QueryBuilderService: Builds dynamic queries with role-specific filters
  • DataFetchingService: Executes optimized database queries
  • FieldMaskingService: Applies role-based field masking

Detailed Process Flow

  1. Role Validation: System validates user role and permissions to ensure proper access control before any data processing
  2. Rules Engine Trigger: Rules engine is triggered to determine data access patterns, filtering criteria, and masking rules based on user role
  3. Query Building: Dynamic query is built using Spring Data JPA with role-specific filters, county restrictions, and access control parameters
  4. Data Fetching: Optimized database query is executed to fetch only the authorized data based on the constructed query parameters
  5. Field Masking Application: Sensitive data is masked according to role-based visibility rules and report type before returning results

Detailed Process Steps

  1. User Request: User submits report generation request with role, date range, and filters
  2. Role Validation: System validates user role and associated permissions
  3. County Filtering: Data is filtered by county location FIRST, then additional filters applied
  4. Data Extraction: Raw timesheet data is extracted from PostgreSQL database
  5. Field Masking: Sensitive data is masked according to role-based visibility rules
  6. Report Generation: Reports are generated in multiple formats (JSON, CSV, XML, PDF)
  7. Event Publishing: Processing events are published to Kafka for audit and monitoring
  8. Response Delivery: Final reports are delivered to requesting user or target system

  • Job status can be monitored via API endpoints
  • Results are stored and can be retrieved when processing is complete
  • Multiple output formats are supported (JSON, CSV, XML, PDF)
